### ‚úÖ Step-by-step explanation:

```js

```

üëâ This line gets the **input box** where you type your task.

```js
const todoList = document.getElementById("todo-list");
```

üëâ This line gets the **empty box** (`div`) where all your tasks will appear.
It saves it in a variable called `todoList`.

---

```js
function addTodo() {
  const value = input.value.trim();
  if (value === "") return;
```

üß† **This is a function** called `addTodo()`. It runs when you click the **"Add"** button.

* `input.value` = whatever you typed.
* `.trim()` removes extra spaces.
* If the input is **empty**, it just stops (does nothing).

---

```js
  const todoItem = document.createElement("div");
  todoItem.className = "todo-item";
```

üß± Creates a new `div` for the new todo task and adds a CSS class `todo-item` to style it.

---

```js
  todoItem.innerHTML = `
    <span>${value}</span>
    <button onclick="removeTodo(this)">Remove</button>
  `;
```

üñäÔ∏è Adds the text of the task inside the new `div` and a "Remove" button next to it.

> `${value}` is the task you typed.

---

```js
  todoList.appendChild(todoItem);
  input.value = "";
}
```

üß∑ Adds the new task to the screen (`todoList`) and **clears** the input box.

---

### üóëÔ∏è Remove Task Function

```js
function removeTodo(button) {
  const todoItem = button.parentElement;
  todoList.removeChild(todoItem);
}
```

* This runs when you click "Remove".
* `button.parentElement` means: find the full todo `div` where the button is.
* `removeChild()` deletes that task from the screen.
